Os códigos abaixo ficam em caixas de texto para não quebrar o HTML e para facilitar copiar e colar.

1. Visão geral do padrão

O padrão adotado é manter uma única classe Importador.cs com métodos separados por entidade, enquanto o Program.cs cria as conexões e entrega os contextos prontos para o importador.

2. Criar o projeto

No Visual Studio, crie um projeto do tipo Console App.

Recomendação: manter o projeto em .NET 8 e os pacotes EF Core em versão 8.x para evitar conflito de versão.

3. Estrutura padrão do projeto

Use uma estrutura simples, separando origem e destino quando houver banco de origem.

Quando a origem for CSV, não é necessário criar a pasta Origem. Nesse caso, o Importador recebe somente o contexto do destino.

10. Boas práticas

  • Use uma única classe Importador.cs com métodos por entidade.
  • Crie os contextos no Program.cs e passe para o Importador pelo construtor.
  • Quando a origem for CSV, passe somente o contexto do destino.
  • Quando a origem for banco, gere Origem e Destino por Scaffold separado.
  • Use AsNoTracking() em consultas de leitura.
  • Use SaveChanges em lote, por exemplo a cada 500 registros.
  • Use ChangeTracker.Clear() após salvar cada lote.
  • Não edite manualmente as classes geradas pelo Scaffold.
